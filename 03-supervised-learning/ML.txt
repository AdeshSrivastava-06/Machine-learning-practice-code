Types of Machine Learning
1. Supervised Learning

Meaning:
In supervised learning, the model is trained on labeled data, where both input and correct output are known.

Data format:
Input → Output (label)

Examples of algorithms:

Linear Regression

Logistic Regression

Decision Tree

KNN

Support Vector Machine (SVM)

Example use cases:

House price prediction

Email spam detection

Student pass/fail prediction

2. Unsupervised Learning

Meaning:
In unsupervised learning, the model is trained on unlabeled data, where no output variable is provided. The model finds patterns or structure in data.

Data format:
Only Input (no output)

Examples of algorithms:
+
K-Means Clustering

Hierarchical Clustering

PCA (Principal Component Analysis)

Isolation Forest

Example use cases:

Customer segmentation

Data compression

Anomaly detection

3. Semi-Supervised Learning

Meaning:
Semi-supervised learning uses a small amount of labeled data and a large amount of unlabeled data to train the model.

Data format:
Few labeled + many unlabeled samples

Examples:
Image classification with few labeled images
Speech recognition systems

Example use cases:
Medical image analysis

Text classification with limited labeled data

4. Reinforcement Learning

Meaning:
In reinforcement learning, the model learns by interacting with the environment and improves its performance based on rewards and penalties.

Learning method:
Trial and error with reward system

Examples:
AlphaGo
Game playing agents
Robotics
Example use cases:

Self-driving cars
Robot navigation
Game AI

  LASSO REGRESSION(L1)

so in the loss function we add some extra term in the 
loss function L = summation(Yi - Yihat)^2 + lamda(|W1| + |W2|) here we take the absolute value not the square
where lamda=hyperparameter(0 to infinity), m=slope the added extra term is also called as shrinkage coefficient

so here the diffrence bet the L2 and L1 is that in L2 when we increase the alpha the line becomes more flat but in L1 when we increase the alpha the line becomes more straight and at alpha=1 it is almost a straight line. This is because as we increase the alpha, the regularization term becomes more significant and the model is forced to fit a simpler line to the data, which can lead to underfitting. In L1 regularization, the coefficients of some features can become exactly zero, which leads to feature selection and a sparser model. In contrast, L2 regularization shrinks the coefficients towards zero but does not set them exactly to zero, resulting in a less sparse model. therefore for complex data use lasso

AUC-ROC(area under curve)

The AUC-ROC measures the entire two-dimensional area underneath the entire
ROC curve from (0,0) to (1,1). AUC provides an aggregate measure of
performance across all possible classification thresholds.
An AUC of 1 indicates that the model has perfect discrimination: it correctly
classifies all positive and negative instances.
•
An AUC of 0.5 suggests the model has no discrimination ability: it is as good
as random guessing.
•
An AUC of 0 indicates that the model is perfectly wrong: it classifies all
positive instances as negative and all negative instances as positive.
•
In practice, AUC values usually fall between 0.5 (random) and 1 (perfect), with
higher values indicating better classification performance.

Like there are multiple features,Like in decision tree, if parallel processing is not performed, then each feature is evaluated individually or sequentially, and then the Gini is calculated. there r 2 types 
[1]Agglomerative cluster

-This algorithm assumes that every data point is a cluster. And then it works by combining each data point as a cluster. 
It stores the information of every cluster in the format of a tree, which is called a dendrogram. It is very useful because the entire data is considered as one cluster, and then there are sub-clusters divided on the basis of the closeness of the point It helps to tell the number of clusters. 

-Agglomerative is mostly used because merging is easier than dividing or breaking , so 

algo - 
[1]Initialize the proximity matrix It contains the distance of each point from all the other points. 
[2]Make each point a cluster. 
[3]Inside a loop 
   Merge the two closest clusters. 
   Update the Proximity Matrix. 
[4]Until only one cluster is left 

Further, agglomerative clustering is divided into further types depending upon how to calculate the distance between two clusters. 
-The distance is also called inter-cluster similarity. 

types - 
[1]Min(single-link)
-Here we will calculate the distance of one point to all the other points, that is, between each possible pair of points, and will select the minimum distance point. 
if our data has lot of outliers noise so it will not work 

[2]Max(complete link)
Same as min, but we will select the point which has the maximum distance. 
outliers and noise can be controlled very well 

[3]Group average
-In between min and max, we calculate the distance of all the possible pairs of points and divide by the number of distances or the number of total distances captured, so it will give the average. 

[4]Ward 
-First, we will calculate the centroid of the two clusters. 
-Then we will calculate the distance of every point from the centroid. 
-dis = C1^2 + C2^2 + ... Cn^2 - A1^2 - A2^2 - .. - An^2 - B1^2 - B2^2 - .. - Bn^2

-Here, C1 and C2 are the distances of each point from the centeroid And a1, a2 ,b1,b2is the distance of each point from its mean, so it means that we are subtracting the distance of the centroid from the variance of the points.
Here, the main factor is to minimize the variance.  
-This is the default value in SK Learner.

    IMBALANCED DATA 

here it means One class is dominating the other class.Suppose there are two classes of pass and fail, and there are a total of 500 data points. 450 students are passed and 50 students are failed, so it is imbalanced. 

problems::-
[1]Bias Our model will focus more on the majority class, leaving behind the minority class, so the result will be in favor of the majority. 
[2]The metrics used will not be reliable. In this case, the accuracy would fail because a class predicting having 90% accuracy on an imbalanced dataset is not reliable. 

imbalanced data is present everywhare in all domains finance,healtcare,manufacturing,customer servies

5 techniques to handle imbalanced data :

[1]Undersampling :-
Here we perform sampling on the majority dataset so that, after sampling, the dataset in the minority and majority is the same, making the data balanced. 

[2]Oversampling
opposite of undersampling we increas our minority class by duplication 

[3]SMOTE(Synthetic Minority Over Sampling Technique ) most popular 
It's an oversampling technique, but the difference is that, instead of duplication of data, which may cause overfitting, it tries to generate new data from the given minority data. 
--It uses interpolation to develop new data points. 

steps:--
[1]Train a KNN model on minority class observations and find each observation's five closest neighbors. 
[2]Select randomly minority class data points. 
[3]Select a neighbor of each data point at random. 
[4]Extract a number between zero and one which is a factor. 
[5]Create the new data point with The following formula: original sample - factor[original sample - neighbor] 
If the original sample is(1,1) and the neighbor is(2,2), then the new generated data point using the above formula will be (1.5,1.5)

we will contiue it until Both the classes become equal. 

DIS-
-not for categorical data 
-compuational complex(KNN is slow)
-Sensitive to outlier
-In the real world, it may be false that the data generated synthetically is not representing the conditions, so it is a big disadvantage. 

[4]ENSEMBLE methods 
Balanced Random Forest - works with imbalanced data 
So in random forest, we train multiple decision rules. And while training, we bootstrap, that is, randomly picked data points. In Balanced Random Forest, the sampling is done in such a way that both the majority class and the minority class counts of data for each decision tree will be the same.
