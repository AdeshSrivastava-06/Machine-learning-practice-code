    HIERARCHICAL CLUSTERING 

there r 2 types 
[1]Agglomerative cluster

-This algorithm assumes that every data point is a cluster. And then it works by combining each data point as a cluster. 
It stores the information of every cluster in the format of a tree, which is called a dendrogram. It is very useful because the entire data is considered as one cluster, and then there are sub-clusters divided on the basis of the closeness of the point It helps to tell the number of clusters. 

-Agglomerative is mostly used because merging is easier than dividing or breaking , so 

algo - 
[1]Initialize the proximity matrix It contains the distance of each point from all the other points. 
[2]Make each point a cluster. 
[3]Inside a loop 
   Merge the two closest clusters. 
   Update the Proximity Matrix. 
[4]Until only one cluster is left 

Further, agglomerative clustering is divided into further types depending upon how to calculate the distance between two clusters. 
-The distance is also called inter-cluster similarity. 

types - 
[1]Min(single-link)
-Here we will calculate the distance of one point to all the other points, that is, between each possible pair of points, and will select the minimum distance point. 
if our data has lot of outliers noise so it will not work 

[2]Max(complete link)
Same as min, but we will select the point which has the maximum distance. 
outliers and noise can be controlled very well 

[3]Group average
-In between min and max, we calculate the distance of all the possible pairs of points and divide by the number of distances or the number of total distances captured, so it will give the average. 

[4]Ward 
-First, we will calculate the centroid of the two clusters. 
-Then we will calculate the distance of every point from the centroid. 
-dis = C1^2 + C2^2 + ... Cn^2 - A1^2 - A2^2 - .. - An^2 - B1^2 - B2^2 - .. - Bn^2

-Here, C1 and C2 are the distances of each point from the centeroid And a1, a2 ,b1,b2is the distance of each point from its mean, so it means that we are subtracting the distance of the centroid from the variance of the points.
Here, the main factor is to minimize the variance.  
-This is the default value in SK Learner. 

How to find the number of clusters using Dendrogram? ??
-First of all, we have to select such a vertical line that is the longest and no other line cuts it. Then we have to cut that horizontal line and see the number of clusters under it, and then we have to use that. 

[2]Divisive clustering 

-It is exactly opposite of Agglomerative. It considers the entire data set as one cluster. 
So again, it will break the biggest cluster into sub-clusters. therefore its divisive
